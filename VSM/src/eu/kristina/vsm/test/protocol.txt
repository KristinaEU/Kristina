Uniform command grammar:
------------------------
{
    "uuid" : <integer>,     	 
    "type" : "turn" | "idle",
    "meta": {
        ...
    },
    "data": {
        ...
    }
}

Example for an idle command:
----------------------------
{
    "uuid" : 23     	 
    "type" : "idle",
    "meta": { 
        "user" :  "Gregor",
        "avatar" : "Kristina",  
    	"language" : "pl", 
        "scenario"  : {
            "type": "medical",
            "name": "babycare"
        } 
    },
    "data" : 
    {
        "faceShift": { 
            "start": 0, 
            "end": 1, 
            "valaro": [0.5, 0.5] 
        }, 
        "blink": true
    }
}

In the idle command the data object represents a BML command.

Example for a turn command:
---------------------------
{
    "uuid":23,     	 
    "type":"turn",
    "meta":{ 
        "user":"Gregor",
        "avatar":"Kristina",  
    	"language":"pl", 
        "scenario":{
            "type":"medical",
            "name":"babycare"
        } 
    },
    "data":{
    	"voice-transcript":"",
    	"language-analysis":"",
        "dialog-management":"",
        "mode-selection": {
            "verbal":"",
            "non-verbal":"",
    	},
    	"language-generation":"",
    	"avatar-turn":"",
    }
}

In the turn command the data object represents a pipeline walkthrough. 
The values of the features within this data object are JSON-encoded 
versions of the data returned by the individual services, which should
not contain unneccessary comments, whitespaces or similar syntax sugar.
